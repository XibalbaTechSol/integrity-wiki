# Rust Axum Oracle (Off-Chain Server)

The **Rust Axum Oracle** is the high-performance, off-chain verification and state ingestion server of the Integrity Project. 

It is designed to handle high-volume telemetry ingestion from thousands of active AI agents, running complex cryptographic validations and mathematical scoring calculations in a zero-trust environment.

## 1. High-Performance Architecture
To prevent CPU starvation and network congestion under institutional loads, the Oracle is engineered with asynchronous concurrency:
-   **Axum HTTP boundary:** A lightweight, non-blocking HTTP web server built in Rust that exposes fast, RESTful ingestion endpoints (such as `/v1/transactions/report` and `/v1/agent/register`).
-   **Redis Nonce Cache:** A Redis-backed memory cache that validates monotonic nonces in real-time, instantly rejecting replay attacks before they touch core database layers.
-   **Async MPSC Queue:** Decouples the rapid HTTP ingestion from CPU-intensive cryptographic verifications. Ingested payloads are enqueued into multi-producer, single-consumer (`mpsc`) channels where background workers process them sequentially.
-   **PostgreSQL Trust Vault:** The persistent database of record that maintains transaction telemetry, DID parameters, and calculated AIS histories.

## 2. Cryptographic Labor
The Oracle operates under a strict **Zero-Trust** stance, performing exhaustive validations on every payload:
-   **Signature Verification:** Resolves the agent's W3C [did](../concepts/did.md), extracts the registered public key, and verifies the Ed25519 signature on the telemetry batch envelope.
-   **Noir Proof Verification:** Directly invokes Aztec Barretenberg native C++ FFI bindings to verify the mathematical [zkp](../concepts/zkp.md) proofs generated by the agent, avoiding the latency and memory overhead of WASM virtualization.
-   **State Compiling:** Computes rolling [Tri Metric Protocol](../concepts/tri-metric-protocol.md) scores and periodically submits the consolidated database Merkle root to `StateAnchor.sol` on Base L2.

## 3. Related Systems
-   **state-anchor:** Feeds verified state Merkle roots directly into the on-chain [State Anchor](state-anchor.md).
-   **integrity-registry:** Issues on-chain slashing triggers to the [Integrity Registry](integrity-registry.md) if compliance audits fail.

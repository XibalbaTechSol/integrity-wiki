# Xibalba Integrity SDK Integration Guide

This guide explains how to integrate the **Xibalba Integrity SDK** into AI agent systems to participate in the [Tri Metric Protocol](tri-metric-protocol.md) reputation ecosystem and enforce pre-execution safety gates.

---

## 1. Installation

### Python
```bash
# Install via pip or uv
pip install xibalba-integrity
```

### Node.js (TypeScript / ES6)
```bash
# Install from npm registry
npm install @xibalba/integrity-sdk
```

## 2. Dynamic State Commitment & Hashing (BCC Core)

AI agents process highly dynamic, non-deterministic inputs. To maintain strict regulatory compliance, agents must cryptographically commit to their intended states before executing mutations.

### 2.1. The `BCCCommitment` Schema
Every commitment generated by the SDK adheres to a strict schema, ensuring tamper-proof verification:

| Field | Type | Description |
| :--- | :--- | :--- |
| `id` | `string (UUID)` | Unique identifier for the commitment. |
| `timestamp` | `integer (Unix)`| ISO 8601 UTC timestamp of commitment generation. |
| `agent_id` | `string` | Cryptographic hardware DID [did](did.md) of the agent. |
| `action_type` | `string` | Categorization of the action (e.g. `hipaa_db_write`, `trade_execution`). |
| `intended_state_hash`| `string (hex)` | Deterministic SHA-256 hash of the sorted canonical JSON state. |
| `opa_policy_id` | `string` | The target identifier of the Open Policy Agent rule to run. |
| `opa_evaluation_result`| `object` | Full JSON decision payload returned by OPA. |
| `provenance_signature` | `string (hex)` | C2PA-style cryptographic signature of the metadata envelope. |

## 3. Cryptographic API Specifications

### 3.1. `commit_action_intent`
Generates a signed commitment of the agent's proposed action state and evaluates it against OPA policies.

```python
def commit_action_intent(
    agent_id: str,
    action_type: str,
    intended_state: dict,
    opa_policy_id: str,
    metadata: dict = None
) -> BCCCommitment:
    """
    Args:
        agent_id (str): Unique hardware-bound DID of the agent.
        action_type (str): Categorization of the proposed action (e.g., "hipaa_db_write").
        intended_state (dict): Predefined JSON schema mapping the proposed modifications.
        opa_policy_id (str): Target OPA compliance ruleset ID.
        metadata (dict, optional): Contextual flags (e.g. "hitl_override": True).

    Returns:
        BCCCommitment: The signed commitment block and OPA evaluation decision.

    Raises:
        CommitmentError: If policy checks fail, or signature generation is rejected.
    """
```

### 3.2. `validate_and_execute`
Performs a final pre-execution check to ensure the real-time execution context matches the cryptographically signed commitment state.

```python
def validate_and_execute(
    commitment: BCCCommitment,
    actual_execution_context: dict,
    action_function: Callable
) -> Any:
    """
    Args:
        commitment (BCCCommitment): The approved pre-execution commitment.
        actual_execution_context (dict): Snapshot of the active environment variables and parameters.
        action_function (Callable): The executable function containing the sensitive task.

    Returns:
        Any: Result of the action_function execution.

    Raises:
        RuntimeValidationError: If actual parameters deviate from the committed state hash.
    """
```

## 4. Drop-In Framework Interceptors

Developers can easily enforce these checks without polluting core logic by wrapping standard LLM client libraries.

### OpenAI Middleware (Python Example)
Our active SDK wraps the standard client in one line, automatically intercepting chat completions to generate and verify commitments:
```python
from integrity_sdk import IntegritySDK
from integrity_middleware import IntegrityOpenAIMiddleware
from openai import OpenAI

sdk = IntegritySDK(local_mode=True, agent_address="0xXibalbaAgent")
middleware = IntegrityOpenAIMiddleware(sdk)

# Wrap the client -- 1-line implementation
client = middleware.wrap_client(OpenAI())

# Generates blind ZK logs and registers telemetry automatically
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "Update record p01: hypertension diagnosis."}]
)
```

### Anthropic Interceptor
```python
from anthropic import Anthropic
from xibalba_integrity import IntegrityClient, AnthropicInterceptor

client = IntegrityClient(config)
interceptor = AnthropicInterceptor(client)

# Wrap the client
anthropic = interceptor.wrap(Anthropic())
```
